# Contributing to xReader

Thanks for your interest in contributing!

- **Full guide:** [docs/contributing.md](docs/contributing.md) ([简体中文](docs/contributing.zh-CN.md))
- **Deployment:** [docs/deployment.md](docs/deployment.md)

## TL;DR

1. One feature/fix per PR; use [Conventional Commits](https://www.conventionalcommits.org/).
2. Tests required — new features need tests, bug fixes need regression tests.
3. `make lint` and `make test` must pass (backend tests need Docker).
4. Ask first before adding DB migrations or new dependencies.
